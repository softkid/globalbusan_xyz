## 📜 바이브 코딩 창세기 — 제독의 3대 서약
본격적인 첫 페이지로 넘어가기 전, 다음 세 가지를 조용히 읽으며 스스로에게 맹세하십시오. 이 서약을 마음속에 새긴 자만이 이 바이블의 진정한 가치를 빼먹을 자격이 있습니다.

* **제1 계명. 나는 내 손으로 직접 코드를 타이핑하지 않을 것이다.**
  (코드를 치는 노동은 AI의 몫이다. 나는 비즈니스 로직을 설계하고 명령만 내리는 지휘관이다.)
* **제2 계명. 나는 빨간 에러 화면을 두려워하거나 피하지 않을 것이다.**
  (에러는 컴퓨터가 고장 난 것이 아니다. 시스템이 나에게 보내는 가장 완벽한 진단서다. 나는 그것을 있는 그대로 AI에게 던질 것이다.)
* **제3 계명. 나는 완벽한 기획보다, 조악하더라도 '오늘 당장 구동되는' 코드를 숭배할 것이다.**
  (서랍 속의 수백억짜리 완벽한 아이디어보다, 오늘 밤 돌아가는 허접한 엑셀 자동화 봇 하나가 네 인생을 바꾼다.)

## 📖 바이브코딩 원론 — 시작하기 전에
코딩의 역사는 70년짜리 삽질이었습니다. 1950년대의 어셈블리 0과 1의 벽돌 쌓기부터 1990년대의 고수준 프로그래밍 언어, 2010년대의 레고 블록식 노코드 플랫폼을 거쳐, **2025년 드디어 '자연어로 명령하여 코드를 통째로 생산해내는' 바이브코딩(Vibe Coding)의 시대**가 열렸습니다.

### 노코드 vs 바이브코딩
노코드가 나쁜 게 아니지만, 결정적인 커스터마이징 순간에 플랫폼의 한계에 부딪히며 소스 코드가 플랫폼에 종속되는 문제가 있습니다. 바이브코딩으로 작성된 코드는 내 하드디스크와 GitHub에 파일 형태로 온전히 존재하기 때문에 플랫폼이 종료되어도 내 소프트웨어는 평생 영속합니다.

## 💡 제0장: AI 수석 개발자 — 안(An) & 로드(Road) 사용 설명서

### 1. 안(An) — 안티그래비티(Anti-gravity)와 일할 때 알아야 할 것
구글 딥마인드 산하의 안티그래비티는 코드를 짜주는 것을 넘어, 본인이 짠 코드를 컴퓨터 터미널에서 스스로 실행해 보고 에러를 만나면 스스로 분석해서 고치는 자율 에이전트(Autonomous Agent)입니다.
* **명령은 "목표" 위주로 하달하십시오:** `"코드가 최종적으로 실패하지 않고 이메일을 수집할 수 있게 끝까지 다 디버깅해서 완성해줘"`라고 목표를 지시하는 게 베스트입니다.
* **기다리십시오:** 백그라운드에서 장애물을 격파할 때 중단하지 말고 믿고 지켜보아야 합니다.
* **컴퓨터 제어권(승인) 부여:** 에이전트가 `run_command` 등 터미널 권한을 요청할 때는 과감하게 승인(Auto-Run)하십시오.

### 2. 로드(Road) — 클로드(Claude)와 일할 때 알아야 할 것
클로드는 세상에서 가장 맥락을 잘 파악하고 거시적 아키텍처 설계를 잘하는 설계자(Architect)입니다.
* **코드는 무조건 통째로 줘라:** 에러를 해결할 때 코드 조각이 아닌 전체 파일(.py, .js 등)을 통째로 올려 맥락 속에서 고치게 하십시오.
* **역할극(Persona)을 부여하라:** 단순 질문 대신 IT 기획자, 전문 개발자 등의 구체적 타이틀을 쥐여줄 때 180도 고품질의 출력을 뽑아냅니다.

### 3. 바이브 코더가 '절대' 하지 말아야 할 것 (DON'Ts)
1. **[절대 금기] 오타나 코드를 내 손으로 직접 수정하는 것:** 키보드에서 완전히 손을 떼십시오. 아주 가벼운 오타 수정조차 AI에게 시키십시오. 사람이 수동으로 고치는 순간 AI와 인간의 코드 인식이 어긋나 전체 빌드가 무너집니다.
2. **[절대 금기] 한 채팅창에서 주제를 이리저리 바꾸는 것:** 뇌 용량(Context Window)이 꼬입니다. 새로운 기능을 만들 때는 무조건 새 창을 열어 리셋해야 합니다.
3. **[절대 금기] 출처 불명의 조각 코드를 마구 섞는 것:** 외부 스니펫은 AI 컨베이어 벨트를 한 번 거쳐 기존 시스템과 통합되도록 해야 충돌이 나지 않습니다.

## 🛠️ 제2장: 초급자를 위한 필수 설치 세팅 (베이스캠프 구축)

### 0단계: 작업 터전 잡기 (폴더의 고립화)
바탕화면이나 C드라이브 한가운데에 빈 작업 폴더 `vibecoding_project`를 만드십시오. 모든 파일은 이 울타리 안에서 격리 관리되어야 경로 에러가 발생하지 않습니다.

### 1단계: 지휘 통제실 설치
* **Node.js 설치:** 컴퓨터가 서버 스크립트를 돌리기 위해 기초 공사를 완료합니다.
* **Claude Code 설치:** 터미널 창을 열고 `npm install -g @anthropic-ai/claude-code`를 입력합니다.
* **안티그래비티 장착:** VS Code를 다운로드받아 좌측 확장 프로그램(Extensions) 탭에서 구글 딥마인드 산하의 최강 자율 에이전트 `Anti-gravity`를 설치합니다.

### 2단계: 파이썬(Python)의 심장부와 절대 보호막 `venv`
1. 구글에 Python Download를 검색해 최신 3.x 버전을 받습니다.
2. **[🚨 필수]** 설치 첫 화면 하단의 **`Add python.exe to PATH`** 체크박스를 무조건 체크하고 설치해야 합니다!
3. 에디터 터미널에 `python -m venv venv`를 쳐서 가상환경 보호막을 만들고, `.\venv\Scripts\activate`로 구동하십시오. 터미널 앞에 `(venv)` 안전마크가 떠오릅니다.

### 3단계: 과금 폭탄을 막는 보급창고 `MASTER.env`
* 보안 비밀번호와 API 키는 소스코드 내부가 아닌, 프로젝트 최상단의 `MASTER.env` 파일에 기록하고 관리해야 합니다.
* `.gitignore` 파일에 `MASTER.env`를 기록하여 GitHub 등에 소중한 개인 자산이 노출되지 않도록 철저히 차단하십시오.

## 💻 제3장: G-Stack(G스택) 방법론과 AI의 뇌 구조

### 1. 마크다운(.md)의 절대성
인공지능은 폰트 서식이 지저분한 워드 파일 대신, 순수 텍스트 뼈대만 남긴 깔끔한 마크다운(.md) 문서를 가장 정확하게 이해합니다. 기획서, 요구사항, 설계 규칙은 항상 마크다운 파일로 격리 작성하십시오.

### 2. G-Stack AI 워크플로우 (역할 분담)
AI 한 명에게 너무 방대한 코딩을 맡기면 금방 맥락을 놓치고 망가집니다. 이를 극복하기 위해:
1. **리서처/기획 봇:** 동향을 분석하여 `plan.md` 기획서를 작성합니다.
2. **보안/감사 봇:** 기획안의 잠재적 취약점이나 규칙 위반을 1차 스캔합니다.
3. **개발 봇:** 기획서 `plan.md` 설계 바탕으로 최종 코드를 무결점 생산합니다.
이렇게 업무를 철저히 쪼개서 에이전트 간 바통 터치 릴레이를 시키는 것이 실리콘밸리식 고수들의 비밀 핵심입니다.

### 3. 영구 기억 장치 `CLAUDE.md` / `.cursorrules`
작업 폴더의 최상단에 이 규칙 정의서를 올려두면, AI가 새로운 대화를 시작할 때마다 이 프로젝트 헌법을 자동으로 백그라운드 참조하여 코드를 규칙대로 일관성 있게 작성합니다.

## 🚀 제11장: B2B 무인 지식창업 파이프라인 (0원에서 100억까지)
1. **1단계 B2B 홈페이지 구축:** Claude를 활용해 고객을 유혹하는 랜딩 페이지를 당일 초고속으로 찍어냅니다.
2. **1.5단계 브릿지 전략:** 첫 번째 클라이언트를 발굴해 0원에 RAG 지식 대시보드를 시범 납품하며 레퍼런스를 만듭니다.
3. **2단계 RAG 지식 연결:** 기업의 고유 매뉴얼이나 문서를 AI에게 주입해 헛소리가 없는 특화 대시보드로 무인화합니다.
4. **3단계 다중 에이전트:** 개발, 보안, 마케팅, CS 부서별 가상 에이전트를 가동해 사람 한 명 뽑지 않는 최정예 무인 비즈니스 제국을 완성합니다.
